# shoppingmall 관리자 화면 — 미구현 기능 및 API 적용 현황

**기준 문서**: `server/docs/API-DOCUMENTATION.md` (v1.0, 2025-03-05)  
**대상 UI**: `shoppingmall/src/app/pages/AdminDashboard.tsx` (`/admin`)

---

## 1. 총괄

| 구분 | 내용 |
|------|------|
| 관리자 화면 API 연동 | **없음**. 모든 데이터는 컴포넌트 내부 `INITIAL_*` 목업이며, 저장/수정/삭제 버튼은 **상태만 바꾸지 않거나** 로컬 state만 갱신하는 수준입니다. |
| 관리자 인증 | **없음**. `/admin` 라우트에 가드·로그인 없음. |
| API 문서에 있는 “관리자 전용” 엔드포인트 | 문서상 **별도 admin 네임스페이스 없음**. 다만 `role`이 `admin`인 계정으로 **일반 REST**를 호출해 상품·브랜드·카테고리 CRUD를 쓰는 방식은 서버 구현에 따라 가능할 수 있음(문서만으로는 확정 불가). |

---

## 2. 메뉴별: 화면 기능 vs API 문서

### 2.1 운영 현황 (`dashboard`)

| 화면에 보이는 것 | API-DOCUMENTATION.md |
|------------------|----------------------|
| 등록 장비 수, 상담 대기 건수, **오늘 방문(1,240 등)**, 자료실 글 수 | **대시보드/방문자/집계 API 없음** (§1~§14 범위 내). |
| 최근 상담 목록 | 컨설팅은 **내 신청 목록**만 §10.3 (`GET /consulting/me`, 인증 필요). **전체 신청 목록(관리자용)은 문서에 없음**. |

**정리**: 운영 지표·전체 상담 피드는 **문서 기준 미충족 + 화면도 목업**.

---

### 2.2 장비/상품 관리 (`products`)

| 화면에 보이는 것 | API-DOCUMENTATION.md |
|------------------|----------------------|
| 상품 목록·신규 등록 폼·수정/삭제 버튼 | **§6 상품**: 목록/상세/인기/브랜드·카테고리별 조회, **§6.9~6.11 생성·수정·삭제** 존재. |
| 실제 연동 | **미적용**. `api-client`·`productApi`를 관리자에서 호출하지 않음. |

**정리**: 서버에 맞춰 연동할 수 있는 후보는 있으나, **관리자 UI는 미구현(목업만)**.

---

### 2.3 메인/추천 관리 (`main_mgmt`)

| 화면에 보이는 것 | API-DOCUMENTATION.md |
|------------------|----------------------|
| 메인 배너(이미지·제목·부제) | **배너/메인 노출 CMS API 없음**. |
| 명장 추천 세트(패키지·포함 장비) | **§9 시뮬레이터 세트**는 “세트” 개념이 유사할 수 있으나, 문서는 **JWT 인증 사용자 기준** 목록/CRUD이며, **메인 배너·추천 패키지 전용 필드와는 불일치**. |

**정리**: 화면 요구사항 대부분이 **문서에 없거나 1:1 매핑 불가**. 기능·API 모두 **미구현**으로 보는 것이 타당.

---

### 2.4 컨설팅 신청 내역 (`consulting`)

| 화면에 보이는 것 | API-DOCUMENTATION.md |
|------------------|----------------------|
| 고객명·연락처·유형·상태 테이블 | **§10.2** `POST /consulting` (신청), **§10.3** `GET /consulting/me` (본인 목록만). |
| 관리자가 **전체 신청** 조회·상태 변경 | **문서에 없음**. |

**정리**: 관리자 테이블은 **목업**. API는 “내 컨설팅”만 문서화되어 있어 **관리자 요구와 불일치**.

---

### 2.5 공지/자료실 관리 (`notices`)

| 화면에 보이는 것 | API-DOCUMENTATION.md |
|------------------|----------------------|
| 공지·강의 등 게시글 CRUD | **게시판/공지/자료실 API 없음** (§1~§14 범위). |

**정리**: **서버 API·연동 모두 문서상 없음**. 화면도 목업.

## 3. API 문서에 있으나 관리자와 무관하거나, 쇼핑몰 앱에서 부분만 사용

다음은 `API-DOCUMENTATION.md`에 정의된 영역입니다. **관리자 대시보드에는 반영되지 않았습니다.**

| 문서 섹션 | 엔드포인트 요약 | shoppingmall 비고 |
|-----------|-----------------|-------------------|
| §2 인증 | register, login, refresh, me, patch me, logout | `AuthContext`는 **목 로그인**(TODO 주석). 관리자 로그인 없음. |
| §3 헬스 | health | 미사용. |
| §4 카테고리 | CRUD | `productApi.getCategories()`는 **CategoryPage** 등에서 사용 가능하나, base URL이 `productApi.ts`에 **하드코드 IP**. 관리자와 무관. |
| §5 브랜드 | CRUD, 목록 | `api/brands.ts`·`useBrands` 존재하나 **BrandsPage는 `constants/brands` 고정**. |
| §6 상품 | 목록·인기·상세·CRUD | **CategoryPage**만 `productApi`로 일부 연동. **MainPage·ProductDetailPage** 등은 대부분 **목 데이터**. |
| §7 장바구니 | 인증 필요 | `CartContext` 등 별도 확인 필요(본 문서는 관리자 중심). |
| §8 위시리스트 | 인증 필요 | 동일. |
| §9 시뮬레이터 | CRUD | **SimulatorPage** 등과 연계 가능하나 관리자 “추천 세트”와는 개념 분리. |
| §10 컨설팅 | 신청·내 목록 | 위 2.4 참고. |
| §11 사용성 서비스 | 신청·내 목록 | 관리자 화면 **미포함**. |
| §12 주문 | 회원/비회원 주문·조회 | 관리자 **주문/결제 관리 UI 없음**. 문서에 **결제(`/payments`) API는 없음**. |

---

## 4. 결론 요약

1. **`AdminDashboard`는 API 미연동 상태**이며, 표시 데이터는 전부 **초기 목업 배열**입니다.  
2. **상품·브랜드·카테고리 CRUD**는 문서상으로는 관리자 구현에 **가장 가까운 후보**이나, UI·토큰·권한 연동은 **아직 없음**.  
3. **배너·추천 패키지·공지/자료실·방문 통계·전체 컨설팅**은 `API-DOCUMENTATION.md`만으로는 **스펙 부재 또는 불충분**이며, 화면도 **플레이스홀더** 수준입니다.  
4. **결제·주문 관리**는 본 API 문서 범위에 **결제 엔드포인트가 없어** 관리자 결제 모니터링과는 **별도 설계**가 필요합니다.

---

## 5. 다음 작업 시 권장 순서 (참고)

1. 관리자 **로그인 + JWT + role 검증** 및 `/admin` **라우트 가드**.  
2. 상품/카테고리/브랜드 중 **문서와 일치하는 DTO**로 CRUD 연동.  
3. 컨설팅·주문·결제는 **관리자용 조회 API가 서버·문서에 추가**된 뒤 화면 설계.  
4. 배너·게시판은 **서버 스펙 추가** 후 CMS 화면 구현.
